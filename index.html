🧠 1. Computational Thinking

A method of problem-solving that involves breaking problems down and using techniques like abstraction, decomposition, pattern recognition, and algorithm design.

Key Concepts:

  • Decomposition: Breaking a complex problem into smaller, more manageable parts.
  • Abstraction: Removing unnecessary details to focus on what’s important.
  • Pattern Recognition: Identifying similarities or patterns in problems.
  • Algorithm Design: Creating step-by-step instructions to solve a problem (e.g., pseudocode, flowcharts).

Sub Programs

Also known as functions or procedures.

Why Use Them?

  • Reuse code
  • Improve readability
  • Make debugging easier

Types:

  • Functions: Return a value (e.g., add(a, b))
  • Procedures: Perform an action but do not return a value

Parameters:

  • Data passed into a subprogram (can be by value or reference)

Search and Sort Algorithms

Linear Search

  • Checks each item in a list until it finds the target
  • Works on unsorted data
  • Time complexity: O(n)

Binary Search

  • Efficient for sorted data
  • Repeatedly divides the search interval in half
  • Time complexity: O(log n)

Bubble Sort

  • Compares adjacent items and swaps if they’re in the wrong order
  • Repeats until the list is sorted
  • Time complexity: O(n²)

Merge Sort

  • Divide-and-conquer algorithm
  • Splits the list into halves, sorts them, then merges them
  • Time complexity: O(n log n)

💾 2. Data

ASCII (American Standard Code for Information Interchange)

  • A character encoding standard
  • Uses 7 bits, representing 128 characters (letters, digits, symbols, control codes)
  • Example: 'A' = 65 in ASCII

Binary

  • Base-2 number system (0s and 1s)
  • Used internally by computers to represent data and instructions

Conversions:

  • Binary → Decimal: Add values of positions where there's a 1
  • Decimal → Binary: Divide by 2 and record remainders

File Size Equations

Units:

  • 1 byte = 8 bits
  • 1 KB = 1024 bytes
  • 1 MB = 1024 KB
  • 1 GB = 1024 MB
  • 1 TB = 1024 GB

Image File Size:

File size = Width × Height × Color Depth

Color depth = bits per pixel (e.g., 24-bit color uses 3 bytes per pixel)

Sound File Size:

File size = Sample rate × Bit depth × Duration × Channels

🌐 3. Networks

Protocols

Rules that govern how devices communicate over a network.

Common Protocols:

  • HTTP/HTTPS: Web browsing (secure with SSL/TLS)
  • FTP: File transfer
  • SMTP/POP3/IMAP: Email transmission/receiving
  • TCP/IP: Core internet protocols (reliable delivery + addressing/routing)

TCP vs UDP

FeatureTCPUDP
ConnectionYes (handshake)No

Transfer Equations

Bandwidth:

The maximum amount of data that can be transferred per second (measured in bps – bits per second)

Transfer Time:

Transfer time = File size / Bandwidth

Make sure units match (convert file size to bits if bandwidth is in bps)

💻 4. Computers

Operating Systems (OS)

Software that manages hardware and software resources

  • Provides user interface and runs applications
  • Functions:
    • Memory management
    • Process scheduling
    • File management
    • Device drivers
    • Security and user access control

Embedded Systems

  • Special-purpose systems built into larger devices
  • Examples: washing machines, car ECUs, smartwatches
  • Usually run real-time operating systems (RTOS)
  • Designed for efficiency, reliability, and low power usage

CPU & Von Neumann Architecture

Components:

  • Control Unit (CU): Directs operations
  • Arithmetic Logic Unit (ALU): Performs calculations and logic
  • Registers: Fast internal storage (e.g., MAR, MDR, PC, ACC)
  • System Bus: Connects CPU, memory, and peripherals

Von Neumann Features:

  • Stores both data and instructions in memory
  • Fetch-Decode-Execute cycle

Bottleneck:

Only one bus between CPU and memory ⇒ limits speed

Diagram: CPU Architecture

┌────────────────────────────┐ │ CPU │ ├────────────┬───────────────┤ │ Control │ Arithmetic │ │ Unit (CU) │ Logic Unit │ │ │ (ALU) │ └────────────┴───────────────┘ ↑ Registers ↑ System Bus ↑ Memory / I/O Devices

Fetch-Decode-Execute Cycle

[Fetch Instruction from RAM] ↓ [Decode Instruction] ↓ [Execute Instruction] ↓ [Repeat]

⚠️ 5. Issues and Impact

Replacement Cycle

Refers to how often technology (like computers, phones) is replaced

  • Driven by planned obsolescence, performance improvements, marketing

Environmental Impact:

  • E-waste
  • Resource depletion (rare metals)
  • Energy consumption

How Malware Works

Types of Malware:

  • Virus: Attaches to programs/files, spreads when executed
  • Worm: Self-replicating, spreads across networks
  • Trojan: Disguised as legitimate software
  • Spyware: Secretly gathers user info
  • Ransomware: Encrypts files and demands payment
  • Adware: Displays unwanted ads

Protection:

  • Antivirus software
  • Firewalls
  • Regular updates
  • Avoid suspicious links/downloads
  • Strong passwords and 2FA
